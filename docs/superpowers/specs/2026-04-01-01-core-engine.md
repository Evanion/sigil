# Spec 01: Core Engine

> Sub-spec of the [Agent Designer PDR](2026-04-01-agent-designer-design.md)

## Overview

The core design engine — a pure-logic Rust library crate with no I/O and no system dependencies. Must compile to both native and WASM without changes.

## Responsibilities

- **Document model** — design tree with typed nodes (frame, rectangle, ellipse, path, text, image, component-instance, group)
- **Node properties** — id, type, name, transform (position, rotation, scale), style (fill, stroke, opacity, blend mode, effects), constraints, children
- **Tree operations** — add, remove, rearrange, transform, group/ungroup nodes
- **Component system** — define components from node subtrees, instantiate with overrides
- **Constraint/layout engine** — flex-based auto-layout on frames, pinning constraints for absolute positioning
- **Pen tool geometry** — bezier path creation, editing, boolean operations on paths
- **Design token model** — token definitions following W3C Design Tokens Format, inheritance resolution across workfiles
- **Token/component promote/demote** — move definitions between workfiles in the hierarchy, conflict detection
- **Diffing and patching** — operation-based undo/redo system
- **Serialization** — to/from the git-diffable JSON file format (one file per page/component, tokens grouped by category)

## Constraints

- No I/O, no filesystem, no networking
- No system dependencies that prevent WASM compilation
- All operations are deterministic and side-effect-free
- Stable node IDs across serialization round-trips

## File Format

Defined in the PDR: JSON files, one per page/component, tokens grouped by category. The core crate owns the serialization/deserialization logic and the schema.
