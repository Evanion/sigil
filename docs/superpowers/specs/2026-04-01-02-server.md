# Spec 02: Server

> Sub-spec of the [Agent Designer PDR](2026-04-01-agent-designer-design.md)

## Overview

The application shell — an Axum-based HTTP server that serves the frontend SPA, manages WebSocket connections for real-time state sync, handles file I/O, and discovers/resolves the workfile hierarchy.

## Responsibilities

- **Serve frontend SPA** — static file serving of the built frontend assets
- **WebSocket state sync** — accept connections from the frontend, receive intent-based operations, apply them through the core engine, broadcast updated state
- **File I/O** — read/write workfiles (`.sigil/` directories) to disk
- **Workfile discovery** — scan from a mount point, find all `.sigil/` directories, resolve inheritance hierarchy by walking parent directories
- **Asset management** — store and serve images, fonts from workfile asset directories
- **Port configuration** — configurable via `PORT` env var or `--port` CLI flag, default 4680

## State Sync Model

Human and agent edits flow through the same operation pipeline:
1. Frontend sends operations via WebSocket; MCP tools invoke operations directly on core
2. Server applies operations through the core engine
3. Server broadcasts updated state to all connected clients
