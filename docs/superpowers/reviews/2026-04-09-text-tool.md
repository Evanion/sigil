# Review Findings — Text Tool (Spec 11b)

**Branch:** feature/text-tool
**Date:** 2026-04-09
**Reviewers:** Architect, Security, Backend, Logic, Compliance, Data, Frontend, A11y, UX (9 agents)

---

## Critical

### RF-001 — `onComplete()` immediately deletes newly-created text nodes
- **Source:** Logic, UX
- **Location:** `text-tool.ts:143-145`, `Canvas.tsx:248-257`
- **Issue:** `onComplete()` switches tool to "select" immediately after `onEditRequest(uuid)`. The tool-change effect calls `commitAndCloseOverlay()`, which reads empty content and calls `deleteNode()` — the user never gets a chance to type.
- **Fix:** Remove `onComplete()` from text tool. Defer tool switch to `commitAndCloseOverlay()`.
- **Status:** open

### RF-002 — `TextStyle` validation bypass via `#[derive(Deserialize)]`
- **Source:** BE, Security, Data
- **Location:** `crates/core/src/node.rs:181`
- **Issue:** `TextStyle` uses `#[derive(Deserialize)]` with public fields. Workfile load and `CreateNode` bypass font_size range, font_family length, and float validation. CLAUDE.md §11 "No Derive Deserialize on Validated Types".
- **Fix:** Extract `validate_text_style()` in validate.rs. Call from `validate_node_kind()` for Text nodes. Consider custom Deserialize impl long-term.
- **Status:** open

### RF-003 — MCP transport has no text style mutation tools
- **Source:** Architect
- **Location:** `crates/mcp/src/tools/nodes.rs`
- **Issue:** GraphQL has 10 text paths; MCP has zero. Agents cannot set text content or style. Violates CLAUDE.md §1 "Agents and humans see each other's changes" and §11 "Validation Must Be Symmetric Across All Transports".
- **Fix:** Defer to follow-up PR with tracking issue. MCP text tools require a spec discussion.
- **Status:** open

### RF-004 — No keyboard entry into text editing mode
- **Source:** A11y
- **Location:** `Canvas.tsx:362-380`
- **Issue:** Double-click is the only way to enter text edit mode. No Enter/F2 keyboard equivalent. Violates WCAG 2.1.1 and CLAUDE.md §11 "Pointer-Only Operations Must Have Keyboard Equivalents".
- **Fix:** Add Enter key handler that opens text overlay when a text node is selected.
- **Status:** open

### RF-005 — Tooltip `as="span"` removes keyboard semantics (PRE-EXISTING)
- **Source:** A11y
- **Location:** `frontend/src/components/tooltip/Tooltip.tsx:27`
- **Issue:** Kobalte Trigger override with `as="span"`. Not introduced by this PR.
- **Fix:** Track separately. Not blocking for this PR.
- **Status:** wont-fix (pre-existing, separate issue)

## High

### RF-006 — `CreateNode::validate` skips TextStyle validation
- **Source:** BE, Security
- **Location:** `crates/core/src/commands/node_commands.rs:32-49`
- **Issue:** `CreateNode::validate` has no arm for `NodeKind::Text` — text style fields are not validated at creation time.
- **Fix:** Add `NodeKind::Text` arm calling `validate_text_style()`.
- **Status:** open

### RF-007 — Missing constant enforcement tests
- **Source:** BE
- **Location:** `validate.rs`, `text_style_commands.rs`
- **Issue:** No `test_<constant>_enforced` tests for MIN_FONT_SIZE, MAX_FONT_SIZE, MIN_FONT_WEIGHT, MAX_FONT_WEIGHT, MAX_FONT_FAMILY_LEN.
- **Fix:** Add 5 enforcement tests.
- **Status:** open

### RF-008 — `kind.content` path: no length check before allocation
- **Source:** Security
- **Location:** `crates/server/src/graphql/mutation.rs:369-384`
- **Issue:** Content string is fully allocated before `SetTextContent::validate` rejects oversized content.
- **Fix:** Add early-out length check before `serde_json::from_value`.
- **Status:** open

### RF-009 — Font family has no character denylist
- **Source:** Security
- **Location:** `validate.rs` (absent)
- **Issue:** No validation for CSS-significant characters (`'`, `"`, `;`, `{`, `}`, `\`) in font_family. Defense-in-depth gap.
- **Fix:** Add `validate_font_family()` with control char + CSS denylist.
- **Status:** open

### RF-010 — Overlay event listeners not cleaned up
- **Source:** Logic, FE, Compliance
- **Location:** `Canvas.tsx:207-225`
- **Issue:** Three `addEventListener` calls (input, blur, keydown) on overlay element not removed on destroy. CLAUDE.md §11 violation.
- **Fix:** Store handler refs, remove in `commitAndCloseOverlay()` before `destroy()`.
- **Status:** open

### RF-011 — `commitAndCloseOverlay` missing try-finally
- **Source:** Logic
- **Location:** `Canvas.tsx:185-196`
- **Issue:** If `store.deleteNode` or `store.setTextContent` throws, `activeOverlay`/`editingUuid` are never cleared, causing double-commit on next close.
- **Fix:** Wrap in try-finally, reset state in finally block.
- **Status:** open

### RF-012 — Underline offset doesn't scale with font size
- **Source:** Logic
- **Location:** `renderer.ts:173-189`
- **Issue:** Underline at hardcoded `+2px` while strikethrough uses proportional `fontSize * 0.3`.
- **Fix:** Use proportional offset: `fontSize * 0.1` or similar.
- **Status:** open

### RF-013 — `setTextStyle` accepts `unknown` value
- **Source:** FE
- **Location:** `document-store-solid.tsx:98,898`
- **Issue:** No type safety between field name and value type. A caller can pass mismatched types.
- **Fix:** Define `TextStylePatch` discriminated union type.
- **Status:** open

### RF-014 — Word-wrap O(W*L) performance
- **Source:** Data
- **Location:** `text-measure.ts:183-185`
- **Issue:** `measureText(candidate)` re-measures entire growing line per word. Should pre-measure space and use scalar addition.
- **Fix:** Pre-measure space char, replace `measureText(candidate)` with `currentWidth + spaceWidth + wordWidth`.
- **Status:** open
