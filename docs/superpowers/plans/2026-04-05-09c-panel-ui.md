# Properties Panel UI (Plan 09c)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single-schema DesignPanel with a sub-tabbed panel (Layout | Appearance | Effects) containing corner radius/constraints editing, fill/stroke list management with DnD reorder and ColorPicker integration, effects list with per-type inline editors, and full keyboard accessibility.

**Architecture:** DesignPanel becomes a sub-tab container using the same tab pattern as TabRegion. Layout tab extends the existing SchemaPanel with new schema entries + MUTATION_MAP handlers. Appearance and Effects tabs are custom panel components because list management (add/remove/reorder with DnD, inline color editing via ColorPicker popover) exceeds the generic schema renderer's capabilities. Fill/Stroke/Effect rows are individual components with DnD handles via dnd-kit-solid.

**Tech Stack:** Solid.js, dnd-kit-solid (existing), Kobalte (Select, Popover), ColorPicker (Plan 09b), lucide-solid icons

## Scope

**In scope:**
- DesignPanel rewrite with Layout | Appearance | Effects sub-tabs
- Layout tab: corner radius (4 inputs + link toggle, rectangle only), constraints (2 selects)

- DnD reorder for fill/stroke/effect lists via dnd-kit-solid
- Keyboard equivalents: Alt+Arrow reorder, Delete remove, H toggle visibility
- Storybook stories for all new components

## Task 1: DesignPanel sub-tabs + Layout schema extensions

Rewrite DesignPanel from a single SchemaPanel into a sub-tabbed container. Extend the design schema with corner radius and constraints sections. Add MUTATION_MAP entries for the new fields.

**Files:**
- Modify: `frontend/src/panels/DesignPanel.tsx`
- Create: `frontend/src/panels/DesignPanel.css`
- Modify: `frontend/src/panels/schemas/design-schema.ts`
- Modify: `frontend/src/panels/SchemaPanel.tsx` (MUTATION_MAP)

The DesignPanel should render 3 sub-tabs (Layout, Appearance, Effects) using simple buttons + conditional rendering (not the full TabRegion system, since these are sub-tabs within a panel). The Layout tab renders the existing SchemaPanel with the extended schema. Appearance and Effects tabs are placeholder divs initially — they'll be built in Tasks 2-5.

The schema extension adds:
- Corner Radius section (when: "rectangle"): 4 number fields for `kind.corner_radii[0-3]`, plus a link toggle
- Constraints section: 2 select fields for horizontal/vertical

MUTATION_MAP additions:
- `corner_radii`: calls `store.setCornerRadii(uuid, [r0, r1, r2, r3])`
- `constraints.`: calls `store.setConstraints(uuid, { ...node.constraints, [field]: value })`

Note: `setConstraints` doesn't exist in the store yet (it was in the core commands but no GraphQL mutation or store method). For now, add just the MUTATION_MAP entry — it will be wired when setConstraints is added. Focus the corner_radii handler since that's fully wired.

## Task 2: FillRow component

A single fill item row with: drag handle, color swatch (opens ColorPicker), fill type label, opacity, visibility toggle, remove button.

**Files:**
- Create: `frontend/src/panels/FillRow.tsx`
- Create: `frontend/src/panels/FillRow.css`

Props: `fill: Fill`, `index: number`, `onUpdate: (index, fill) => void`, `onRemove: (index) => void`, `isFocused: boolean`, `onFocusRow: (index) => void`

The color swatch is a button that triggers a ColorPicker popover. When the color changes, construct a new Fill with the updated color and call `onUpdate`. The opacity field reads from `fill.color.value.a` for solid fills. Visibility toggle and remove button use the same pattern as TreeNode (tabindex=-1, keyboard via parent).

## Task 5: EffectCard + EffectsPanel

EffectCard renders a single effect with a type dropdown and per-type fields. EffectsPanel manages the effects list.

**Files:**
- Create: `frontend/src/panels/EffectCard.tsx`
- Create: `frontend/src/panels/EffectCard.css`
- Create: `frontend/src/panels/EffectsPanel.tsx`
- Create: `frontend/src/panels/EffectsPanel.css`

EffectCard props: `effect: Effect`, `index: number`, `onUpdate: (index, effect) => void`, `onRemove: (index) => void`, `isSelected: boolean`

Effect type dropdown: Drop Shadow, Inner Shadow, Layer Blur, Background Blur. Changing type preserves compatible fields (shadow→shadow keeps color/offset/blur/spread, shadow→blur only keeps blur radius as radius).

Per-type fields:
- Drop Shadow / Inner Shadow: color swatch, X offset, Y offset, blur, spread
- Layer Blur / Background Blur: radius only

EffectsPanel: "+" adds a default DropShadow. DnD reorder. Keyboard Alt+Arrow/Delete.

Wire into DesignPanel's "Effects" sub-tab.
