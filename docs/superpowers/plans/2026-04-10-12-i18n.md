# i18n Infrastructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up i18next internationalization infrastructure and migrate all hardcoded user-facing strings to translation keys — English only for v1, with infrastructure ready for community translations.

**Architecture:** i18next core with `@mbarzda/solid-i18next` for Solid.js reactive integration. Translation strings split into namespace JSON files (common, tools, panels, a11y). Kobalte's locale is synced with i18next's language. Strings migrated file-by-file in priority order: aria-labels first, then panel headers, then button labels.

**Tech Stack:** i18next, @mbarzda/solid-i18next, TypeScript, Solid.js, Vitest

- [ ] **Step 2: Create i18n initialization module**

Create `frontend/src/i18n/index.ts`:

```typescript
import i18next from "i18next";

import commonEn from "./locales/en/common.json";
import toolsEn from "./locales/en/tools.json";
import panelsEn from "./locales/en/panels.json";
import a11yEn from "./locales/en/a11y.json";

/** Locale persistence key in localStorage. */
const LOCALE_STORAGE_KEY = "sigil-locale";

/** Read the persisted locale or fall back to browser language, then to English. */
function getInitialLocale(): string {
  try {
    const stored = localStorage.getItem(LOCALE_STORAGE_KEY);
    if (stored) return stored;
  } catch {
    // localStorage may be unavailable (private browsing, SSR)
  }
  return navigator.language ?? "en";
}

/** Persist the selected locale to localStorage. */
export function persistLocale(locale: string): void {
  try {
    localStorage.setItem(LOCALE_STORAGE_KEY, locale);
  } catch {
    // Silently ignore — localStorage may be unavailable
  }
}

/** Initialize i18next with all English namespaces pre-loaded. */
export async function initI18n(): Promise<typeof i18next> {
  await i18next.init({
    lng: getInitialLocale(),
    fallbackLng: "en",
    defaultNS: "common",
    ns: ["common", "tools", "panels", "a11y"],
    interpolation: {
      escapeValue: false, // Solid.js handles XSS via JSX
    },
    resources: {
      en: {
        common: commonEn,
        tools: toolsEn,
        panels: panelsEn,
        a11y: a11yEn,
      },
    },
  });
  return i18next;
}

export { i18next };
```

- [ ] **Step 3: Add a test for i18n initialization**

Create `frontend/src/i18n/__tests__/i18n.test.ts`:

```typescript
import { describe, it, expect } from "vitest";
import i18next from "i18next";
import { initI18n } from "../index";

describe("i18n initialization", () => {
  it("should initialize with English as default", async () => {
    await initI18n();
    expect(i18next.language).toMatch(/^en/);
  });

  it("should load common namespace", async () => {
    await initI18n();
    expect(i18next.t("common:ok")).toBe("OK");
    expect(i18next.t("common:cancel")).toBe("Cancel");
  });

  it("should load tools namespace", async () => {
    await initI18n();
    expect(i18next.t("tools:select.label")).toBe("Select");
  });

  it("should support interpolation", async () => {
    await initI18n();
    expect(i18next.t("a11y:page.created", { name: "Page 2" })).toBe("Created Page 2");
  });

  it("should fall back to English for unknown locale", async () => {
    await initI18n();
    await i18next.changeLanguage("xx");
    expect(i18next.t("common:ok")).toBe("OK");
  });
});
```
