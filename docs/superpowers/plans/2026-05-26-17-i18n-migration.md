# Plan 17 — i18n Migration + CI Guard

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the i18n migration — extract every remaining hardcoded user-facing string in `frontend/src` into the existing `@mbarzda/solid-i18next` framework (adding the corresponding keys to en/fr/es locale JSON files), add an ESLint rule + locale-parity script that block regressions, and lock the gate so future PRs cannot reintroduce hardcoded strings.

**Architecture:** The library + scaffolding + three populated locales are already shipped (verified during plan-writing: `@mbarzda/solid-i18next` + `i18next` installed, `TransProvider` wired in `App.tsx`, `i18nInstance` loads 4 namespaces × 3 locales, `localStorage`-backed locale persistence with `navigator.language` fallback). The plan adds (a) `eslint-plugin-i18next` configuration, (b) a Node script verifying en/fr/es key parity, (c) the actual extraction of remaining hardcoded literals into `t("namespace:key")` calls, and (d) per-locale rendering smoke tests.

**Tech Stack:** TypeScript (strict), Solid.js, `@mbarzda/solid-i18next`, `i18next`, ESLint 9 flat config with `eslint-plugin-i18next`, Node 22, Vitest.

**Branch:** `feature/i18n-spec-17` (worktree at `.worktrees/feature/i18n-spec-17`, based on `main`'s post-PR-#65 state).

## Pre-work: confirmed context

The spec at `docs/superpowers/specs/2026-05-26-17-i18n-migration.md` describes the design intent but overstates the remaining work. The starting state on this branch is:

- **Library:** `@mbarzda/solid-i18next` 1.4.1 + `i18next` 26.0.4 in `frontend/package.json`.
- **Init:** `frontend/src/i18n/index.ts` exports `i18nInstance` that loads all four namespaces (`common`, `tools`, `panels`, `a11y`) for all three locales (`en`, `fr`, `es`).
- **Detection:** `getInitialLocale()` reads `localStorage["sigil-locale"]` first, then `navigator.language`, fallback to `"en"`. Uses `load: "languageOnly"` to normalize BCP-47 subtags.
- **Persistence:** `persistLocale(lng)` writes to localStorage; the i18next `languageChanged` event handler keeps localStorage in sync.
- **TransProvider:** wired in `frontend/src/App.tsx` around `<AppShell />`.
- **Locale files:** `frontend/src/i18n/locales/{en|fr|es}/{common,tools,panels,a11y}.json` — 311 lines per locale, French and Spanish are real translations (e.g., en `"cancel"` → fr `"Annuler"` → es `"Cancelar"`).
- **Test utility:** `frontend/src/test-utils/i18n.ts` exports `createTestI18n()` for component tests.
- **Adoption today:** 24 files use `useTransContext` / `t()`. ~36 files still contain hardcoded `aria-label="..."` strings; an inventory pass (Task 3) produces the authoritative list.

**What this plan does NOT do** (because already shipped):
- Pick an i18n library.
- Add language detection.
- Add locale persistence.
- Generate fr/es locale drafts.
- Add `_meta` provenance blocks (translations look reasonable; treating them as in-place is cleaner than adding metadata + a future "removal" task).

```javascript
// frontend/scripts/check-locale-parity.mjs
/**
 * Verifies that fr/ and es/ locale JSON files have the same key tree
 * as the corresponding en/ files. Top-level keys starting with "_"
 * (e.g., "_meta") are skipped — locales may legitimately diverge on
 * provenance metadata.
 *
 * Exit code 0 if all match; 1 if any divergence found.
 *
 * Usage: node frontend/scripts/check-locale-parity.mjs
 */

import { readFileSync } from "node:fs";
import { join, dirname } from "node:path";
import { fileURLToPath } from "node:url";

const __dirname = dirname(fileURLToPath(import.meta.url));
const LOCALES_DIR = join(__dirname, "..", "src", "i18n", "locales");
const NAMESPACES = ["common", "tools", "panels", "a11y"];
const LOCALES = ["en", "fr", "es"];

/** Recursively collect dotted key paths from a nested object,
 *  skipping any top-level key beginning with "_". */
function collectKeys(obj, prefix = "", isTopLevel = true) {
  const keys = [];
  for (const [k, v] of Object.entries(obj)) {
    if (isTopLevel && k.startsWith("_")) continue;
    const fullKey = prefix ? `${prefix}.${k}` : k;
    if (v !== null && typeof v === "object" && !Array.isArray(v)) {
      keys.push(...collectKeys(v, fullKey, false));
    } else {
      keys.push(fullKey);
    }
  }
  return keys;
}

/** Compare two locale trees; returns missing + extra keys in `b`
 *  relative to `a`. Skips top-level `_*` keys. */
export function compareLocaleTrees(a, b) {
  const aKeys = new Set(collectKeys(a));
  const bKeys = new Set(collectKeys(b));
  const missing = [...aKeys].filter((k) => !bKeys.has(k));
  const extra = [...bKeys].filter((k) => !aKeys.has(k));
  return { missing, extra };
}

function loadLocaleNamespace(locale, ns) {
  const path = join(LOCALES_DIR, locale, `${ns}.json`);
  try {
    return JSON.parse(readFileSync(path, "utf-8"));
  } catch (err) {
    throw new Error(`Failed to load ${path}: ${err.message}`);
  }
}

function main() {
  let totalFailures = 0;
  const failures = [];

  for (const ns of NAMESPACES) {
    const enTree = loadLocaleNamespace("en", ns);
    for (const locale of LOCALES.filter((l) => l !== "en")) {
      const otherTree = loadLocaleNamespace(locale, ns);
      const { missing, extra } = compareLocaleTrees(enTree, otherTree);
      if (missing.length > 0 || extra.length > 0) {
        totalFailures++;
        failures.push({ namespace: ns, locale, missing, extra });
      }
    }
  }

  if (totalFailures === 0) {
    console.log("✓ All locale key trees match en/.");
    process.exit(0);
  }

  console.error("✗ Locale parity check failed:");
  for (const { namespace, locale, missing, extra } of failures) {
    console.error(`\n  Namespace: ${namespace}, Locale: ${locale}`);
    if (missing.length > 0) {
      console.error(`    Missing from ${locale}/${namespace}.json:`);
      for (const k of missing) console.error(`      - ${k}`);
    }
    if (extra.length > 0) {
      console.error(`    Extra in ${locale}/${namespace}.json (not in en):`);
      for (const k of extra) console.error(`      - ${k}`);
    }
  }
  process.exit(1);
}

// Run main() only when invoked directly, not when imported by tests.
if (import.meta.url === `file://${process.argv[1]}`) {
  main();
}
```

```yaml
  i18n-allowlist-rationale:
    name: i18n Allowlist Rationale
    runs-on: ubuntu-latest
    needs: detect-changes
    if: needs.detect-changes.outputs.frontend == 'true'
    steps:
      - uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6
      - name: Verify every i18next/no-literal-string disable has a rationale
        run: |
          set -euo pipefail
          # Find every eslint-disable line that targets i18next/no-literal-string.
          # Each match must have an "i18n-allow:" comment within +/- 1 line, or
          # an "// i18n-allow:" trailing comment on the same line as the disable.
          matches=$(grep -rEn 'eslint-disable(-next-line)?[^\n]*i18next/no-literal-string' frontend/src --include='*.ts' --include='*.tsx' || true)
          if [ -z "$matches" ]; then
            echo "No i18next/no-literal-string disables found — nothing to check."
            exit 0
          fi
          bad=""
          while IFS= read -r line; do
            file=$(echo "$line" | cut -d: -f1)
            lineno=$(echo "$line" | cut -d: -f2)
            # Look at the line itself and the line above for "i18n-allow:".
            window=$(sed -n "$((lineno - 1)),${lineno}p" "$file")
            if ! echo "$window" | grep -q 'i18n-allow:'; then
              bad="$bad\n$file:$lineno"
            fi
          done <<< "$matches"
          if [ -n "$bad" ]; then
            echo "::error::eslint-disable for i18next/no-literal-string lacks an // i18n-allow: rationale comment. Add a rationale or restructure to avoid the disable."
            echo -e "$bad"
            exit 1
          fi
          echo "All i18next/no-literal-string disables have rationale comments."
```
